**Premise: you already have a second brain ‚Äî assign roles, don't build a new
system.** kolu's existing stores:

- **GitHub Issues** ‚Äî lightweight, living nodes.
- **In-repo docs** ‚Äî substantial, structured artifacts.
- **The blog** ‚Äî the public stream.
- **The Code tab** ‚Äî renders + annotates both.

## The one routing rule

> *Substantial, structured artifact ‚Äî or lightweight, transient node?*

| Substantial ‚Üí an **Atlas note** | Lightweight ‚Üí a **GitHub Issue** |
| --- | --- |
| Plans, designs, research, reviews, retros, history | Bugs, tasks, roadmap items, quick questions |

- **Maturity is a tag, not a divider** ‚Äî `seedling` ‚Üí `budding` ‚Üí `evergreen`; never a location, never a routing axis.
- **Living ‚âÝ frozen** ‚Äî a plan like `remote-terminals` evolves for months and still lives in the Atlas.
- **The boundary blurs by design** ‚Äî a concept can hold both a note and an issue.
- **Extract, don't sync** ‚Äî when an issue thread becomes the source of truth, lift its summary into a note.

## The map

| Surface | Where | Role |
| --- | --- | --- |
| **Public** | the blog (kolu.dev) + per-release changelog | outward-facing; one post per release |
| **Atlas** | `docs/atlas/` ‚Üí `docs/atlas/dist/` | the working brain; markdown/MDX notes, internal-only |

- **History isn't a third place** ‚Äî it's the Atlas over time: a settled note is just `evergreen`, git is the history, the changelog is the release artifact.

## Where it lives: a self-contained Astro project

`docs/atlas/` is its **own little Astro project** ‚Äî decoupled from the public
`website/` (different audience + cadence), **not published anywhere**.

- **Author markdown/MDX, get HTML** ‚Äî write `.md`/`.mdx` + frontmatter; Astro renders.
- **One layout + theme** ‚Äî no per-file CSS (the old HTML notes duplicated ~76 KB).
- **Generated index** from frontmatter ‚Äî no hand-curated map (see *Navigation*).
- **Self-contained committed output** ‚Äî each `dist/<slug>.html` inlines its styles and cross-links with relative hrefs ‚Üí previews in the Code tab, no server.
- **`draft: true`** hides a half-baked note from the index but keeps it on disk for agents.

## Format ‚Äî markdown prose + MDX components

- **Prose in markdown** ‚Äî the default.
- **`.mdx` when a note needs more** ‚Äî import typed Astro/TS components, use them inline. No raw HTML, no per-note CSS.

| Why markdown for prose | Evidence |
| --- | --- |
| Far fewer tokens ‚Äî paid on every agent read | Cloudflare: ~80% fewer tokens md vs html |
| The CLIs kolu runs prefer it | Claude Code & OpenCode send `Accept: text/markdown` |
| Renders where it matters | github.com renders `.md`; the Code tab renders it (#1093) |

### Bring your own component

The shared kit isn't a cage ‚Äî a session can define or import its own, in order
of friction (all three are live in *this* note):

1. **Inline in this file** ‚Äî an `export const` at the top of the `.mdx` (JSX
   rules: `style={{‚Ä¶}}`, `className`). Defined here as `Spark`, used live:
   <Spark>inline</Spark>.
2. **Co-located `.astro` file** ‚Äî a real component (scoped `<style>`, typed
   props, plain HTML ‚Äî no JSX gotcha) imported just for this note from
   `src/components/second-brain/Stamp.astro`: <Stamp label="note-local" />.
3. **Raw inline markup** ‚Äî for a true one-off, drop JSX inline, no named
   component: <span style={{ fontFamily: "ui-monospace, monospace", fontSize: "0.8em", border: "1px dashed var(--rule)", borderRadius: "5px", padding: "0.05em 0.4em", color: "var(--ink-dim)" }}>raw inline JSX</span>.

Each rebuilds with `just atlas::build` and passes `ci::atlas-sync` like the kit.
**Promotion path:** inline ‚Üí co-located ‚Üí shared kit, once it proves reused.

## Navigation ‚Äî generated, not hand-curated

- **Index built from frontmatter** ‚Üí a note can never be unfiled (retires the hand MOC + its `docs-moc` link-gate, kept only while legacy `docs/plans/*.html` exist).
- **Flat, ancestry-free slugs** ‚Äî the filename is a handle, not a path.
- **Index groups by `kind`, shows `maturity`, hides drafts.**
- **A nested ToC** (`<Toc>`) is auto-inserted from each note's headings ‚Äî the box above.
- **Dead-link checking stays a generic linter**, not a bespoke gate.

## Proposals & decisions ‚Äî keep the numbered log

- Value of a decision/RFC log = **stable monotonic id** + **append-only immutability** + **supersede links**.
- Keep proposals as **numbered markdown** (`status: proposed ‚Üí accepted`) ‚Äî already required by `CONTRIBUTING.md`.
- `proposed ‚Üí accepted` is a *lifecycle* (a status field) ‚Äî no separate `docs/decisions/` dir needed.
